# Wardrobe Configurator

Статический фронтенд на JavaScript для 3D-рендера шкафа по YAML-конфигу. Бэкенда нет: базовый конфиг лежит в `public/wardrobe.yaml`, его можно править руками или менять прямо в интерфейсе.

## Запуск

```powershell
npm.cmd install
npm.cmd run dev
```

После запуска откройте адрес, который покажет Vite, обычно `http://127.0.0.1:5173`.

## Конфиг

Главный файл: `public/wardrobe.yaml`.

- `dimensions` задает внешние габариты шкафа в миллиметрах: высота считается от пола до верха крышки, ширина — от левого внешнего бока до правого внешнего бока.
- `boardThickness` задает толщины досок в миллиметрах. `default` используется как запасной вариант, а `sides`, `top`, `bottom`, `shelves`, `dividers`, `back`, `doors` задают толщины для групп деталей.
- У отдельной доски можно указать свою толщину через `thickness`; это работает для элементов в `verticalDividers`, `internal` и `doors`.
- `columns` задает секции слева направо.
- `verticalDividers` задает вертикальные перегородки между секциями, включая неполную высоту через `fromFloor` и `toTop`.
- `rows` задает ряды снизу вверх.
- `doors` задает дверцы. Дверца может занимать несколько ячеек через `rowSpan` и `colSpan`.
- `handle: none` скрывает ручку на фасаде.
- Если ячейка не перекрыта дверцей, она остается открытой.
- `openCells` добавляет подсветку и декоративные предметы в открытые ячейки.
- `internal` добавляет внутренние элементы внутри выбранного диапазона секций: `type: shelf` для полок и `type: rail` для штанги. Для элементов можно задавать `rowSpan`, а высоту внутри выбранного пространства задавать через `heightRatio` от `0` до `1` или `offsetFromTop`.
- `baskets` добавляет сетчатые выдвижные корзины для белья.

По клику на фасад или ручку дверца открывается и закрывается. Для направления открывания используйте `swing: left` или `swing: right`.
